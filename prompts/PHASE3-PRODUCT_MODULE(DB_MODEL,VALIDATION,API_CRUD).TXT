# PHASE 3 - PRODUCT MODULE (DATABASE MODEL + VALIDATION + API CRUD)

Read **AGENT.md** first and follow it STRICTLY.

Follow **PHASE 0 - GLOBAL INSTRUCTIONS** before implementing anything.

Follow the authentication structure implemented in Phase 2.

This phase is ONLY for Product Management.

Do NOT implement:

* Cart
* Checkout
* Orders
* Payment Gateway
* Wishlist
* Reviews
* Real-time stock updates
* Search suggestions
* Product filters

Implement Product Module only.

---

## Objective

Build a complete Product Management module using:

* Next.js 15 App Router
* TypeScript
* MongoDB Atlas
* Mongoose
* Zod
* Next.js API Routes

The Product module should support:

* Create Product
* Get All Products
* Get Product By ID
* Update Product
* Delete Product

# Product Database Schema

Create:

```txt
models/Product.ts
```

Schema:

```ts
{
title:string,

description:string,

price:number,

discount:number,

clothType:string,

brand:string,

size:string,

color:string,

stock:number,

image:string,

createdAt:Date,

updatedAt:Date
}
```

Enable:

```ts
timestamps:true
```

---

## Field Rules

### title

```txt
required

minimum 3 characters

maximum 100 characters
```

Example:

```txt
Men Regular Fit Cotton Shirt
```

---

### description

```txt
required

minimum 20 characters
```

---

### price

```txt
required

must be greater than 0
```

---

### discount

```txt
default 0

minimum 0

maximum 100
```

---

### clothType

Examples:

```txt
Shirt

T-Shirt

Jeans

Hoodie

Jacket

Pants
```

Required.

---

### brand

Examples:

```txt
Nike

Adidas

Puma

Levis
```

Required.

---

### size

Store:

```txt
XS

S

M

L

XL

XXL
```

Store as string.

---

### color

Example:

```txt
Black

White

Blue

Red
```

Store as string.

---

### stock

```txt
required

minimum 0
```

---

### image

Store ONLY:

```ts
image:"https://..."
```

Example:

```txt
https://images.unsplash.com/....
```

Do NOT:

* Upload images

* Use multer

* Store image files

* Store base64 images
