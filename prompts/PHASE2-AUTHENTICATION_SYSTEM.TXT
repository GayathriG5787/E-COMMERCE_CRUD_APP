# PHASE 2 - AUTHENTICATION SYSTEM

Read **AGENT.md** first and follow it STRICTLY.

Follow **PHASE 0 - GLOBAL INSTRUCTIONS** before implementing anything.

This phase is ONLY for Authentication.

Do NOT implement:

* Product CRUD
* Cart functionality
* Admin Dashboard
* Product pages
* Order system
* Real-time features

Implement Authentication only.

---

## Objective

Build a secure JWT Authentication system using:

* Next.js 15 App Router
* TypeScript
* MongoDB Atlas
* Mongoose
* JWT
* bcryptjs
* httpOnly Cookies
* Zod Validation

Authentication must support:

* Register
* Login
* Logout
* Get Current User
* Role based access preparation

---

## Features Included

### User Registration

User can register using:

```txt
Name

Email

Password
```

Requirements:

* Name is required
* Email is required
* Email must be valid
* Password is required
* Password minimum 6 characters

Before creating user:

* Check if email already exists.
* Return proper error if already registered.

Password:

* Hash using bcryptjs.
* Never store plain text passwords.

---

### User Login

User can login using:

```txt
Email

Password
```

Requirements:

* Verify email exists.
* Compare password using bcryptjs.
* Generate JWT after successful login.

Store JWT:

```txt
httpOnly Cookie
```

Do NOT:

* Store JWT in localStorage.
* Store JWT in sessionStorage.
* Expose token to frontend.

---

### User Logout

Logout should:

* Remove authentication cookie.
* Invalidate user session on browser.

Return:

```json
{
"success": true,
"message": "Logged out successfully"
}
```

---

### Get Current User

Create API:

```txt
GET /api/auth/me
```

Return:

```ts
{
id,
name,
email,
role
}
```

Never return:

```txt
password
```

# Cookie Rules

Store token inside:

```txt
httpOnly Cookie
```

Configuration:

```txt
httpOnly: true

secure: production only

sameSite: "strict"

path: "/"
```

Never:

* Store token in localStorage
* Send token in frontend state
* Expose secret
