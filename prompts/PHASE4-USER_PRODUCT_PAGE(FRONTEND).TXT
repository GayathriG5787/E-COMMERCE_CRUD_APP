# PHASE 4 - USER PRODUCT PAGES (FRONTEND)

Read **AGENT.md** first and follow it STRICTLY.

Follow **PHASE 0 - GLOBAL INSTRUCTIONS** before implementing anything.

Use the APIs and Product Model created in previous phases.

This phase is ONLY for User Product UI.

Do NOT implement:

* Cart functionality
* Checkout
* Orders
* Wishlist
* Reviews
* Ratings
* Search autocomplete
* Product filters
* Pagination
* Admin Dashboard
* Real-time features

Implement Product UI only.

---

## Objective

Build user-facing Product Pages using:

* Next.js 15 App Router
* TypeScript
* Tailwind CSS v4
* Shadcn UI
* Axios
* Redux Toolkit

Users should be able to:

* View all products
* View product details
* Navigate between products
* See product information
* See Add To Cart button (UI only)

Do NOT implement cart logic yet.

---

# Pages To Create

Create:

```txt
app/

products/

page.tsx


products/[id]/

page.tsx
```

---

# Products Listing Page

Route:

```txt
/products
```

Purpose:

Display all products in a responsive grid.

Fetch data from:

```txt
GET /api/products
```

---

## Product Grid

Requirements:

Responsive:

```txt
Mobile:

1 column


Tablet:

2 columns


Desktop:

3 or 4 columns
```

Use:

```txt
Tailwind Grid
```

Do not use:

```txt
Bootstrap

CSS Grid libraries
```

# Price Section

Display:

```txt

Discount

Final Price
```

```txt

```

```txt

```

---

# Stock Status

Display:

If:

```txt
stock > 0
```

Show:

```txt
In Stock
```

If:

```txt
stock === 0
```

Show:

```txt
Out of Stock
```

Disable:

```txt
Add To Cart button
```

UI only.
